Synchronizing with a remote location

Once you have imported or exported a project of folder to a remote location the SFTP support remembers the association between the local resources and the remote location. This means that you can use the Synchronize View to move files back-and-forth between the local project and the remote location, as well as keep them synchronized (i.e. upload and download changes).

To create a Synchronization first open the Team Synchronizing perspective, then from the Synchronize View click on the first button to open the Create Synchronization wizard. Select the SFTP synchronizations. On the next page, you can select which resources in your workspace should be included in the synchronization. Only those resources that have been previously imported or exported will be available.

When the synchronization is added to the Synchronize View you can use the context menu to upload and download changes.
